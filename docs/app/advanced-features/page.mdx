# Advanced Features

i18nizer extracts translatable strings from your components and generates standard i18n JSON files. For advanced patterns like pluralization and rich text formatting, you can enhance the extracted translations using the powerful built-in features of i18next and next-intl.

## Pluralization Support

### Overview

Pluralization allows you to handle different text variations based on counts. Both i18next and next-intl support the ICU (International Components for Unicode) message format, which provides a standardized way to handle plurals across different languages.

### Why Use ICU Message Format?

Different languages have different plural rules:
- **English**: 2 forms (one, other)
- **Polish**: 3 forms (one, few, many)
- **Arabic**: 6 forms (zero, one, two, few, many, other)
- **Chinese/Japanese**: 1 form (other)

ICU message format automatically selects the correct form based on the locale.

### Basic Pluralization Example

**Before i18nizer:**
```tsx
function ShoppingCart({ count }) {
  return (
    <div>
      <p>{count} {count === 1 ? 'item' : 'items'} in cart</p>
    </div>
  )
}
```

**After i18nizer extraction:**
```tsx
import { useTranslations } from 'next-intl'

function ShoppingCart({ count }) {
  const t = useTranslations('ShoppingCart')
  
  return (
    <div>
      <p>{t('itemCount', { count })} {t('itemsLabel', { count })}</p>
    </div>
  )
}
```

**Manually enhance with pluralization:**
```tsx
import { useTranslations } from 'next-intl'

function ShoppingCart({ count }) {
  const t = useTranslations('ShoppingCart')
  
  return (
    <div>
      <p>{t('itemsInCart', { count })}</p>
    </div>
  )
}
```

**Translation file (`messages/en/shopping-cart.json`):**
```json
{
  "ShoppingCart": {
    "itemsInCart": "{count, plural, =0 {No items} one {# item} other {# items}} in cart"
  }
}
```

### ICU Plural Syntax

```
{variable, plural, category {text} category {text} ...}
```

**Categories:**
- `=0`, `=1`, `=2` - Exact matches
- `zero` - Zero items (not all languages use this)
- `one` - Singular (typically 1, but varies by language)
- `two` - Dual (some languages like Arabic)
- `few` - Few items (some languages)
- `many` - Many items (some languages)
- `other` - Default/fallback (required)

**Placeholders:**
- `#` - Replaced with the actual count number
- `{variable}` - Any variable passed to the translation

### More Pluralization Examples

#### Example 1: Notifications Counter

```tsx
<p>{t('notificationCount', { count: unreadCount })}</p>
```

```json
{
  "notificationCount": "{count, plural, =0 {No new notifications} one {# new notification} other {# new notifications}}"
}
```

#### Example 2: Days Remaining

```tsx
<span>{t('daysRemaining', { days: remainingDays })}</span>
```

```json
{
  "daysRemaining": "{days, plural, =0 {Expires today} =1 {# day remaining} other {# days remaining}}"
}
```

#### Example 3: File Upload

```tsx
<p>{t('filesSelected', { count: selectedFiles.length })}</p>
```

```json

```

```tsx

```

## Rich Text Formatting

### Overview

Rich text formatting allows you to include JSX elements (like links, bold text, or custom components) within your translated strings. This is essential when you need to maintain the structure and styling of text while keeping it translatable.

### Why Use Rich Text?

- **Keep translations atomic**: Don't split sentences just to add a link
- **Translator-friendly**: Translators see the full sentence with placeholders
- **Flexibility**: Different languages might place links in different positions
- **Type safety**: Components passed to rich text are type-checked

### Rich Text with next-intl

#### Basic Link Example

**Before i18nizer:**
```tsx
function TermsNotice() {
  return (
    <p>
      By clicking Sign Up, you agree to our <a href="/terms">Terms of Service</a>
    </p>
  )
}
```

**After i18nizer extraction:**
```tsx
import { useTranslations } from 'next-intl'

function TermsNotice() {
  const t = useTranslations('TermsNotice')
  
  return (
    <p>
      {t('byClickingSignUp')} <a href="/terms">{t('termsOfService')}</a>
    </p>
  )
}
```

**Manually enhance with rich text:**
```tsx
import { useTranslations } from 'next-intl'

function TermsNotice() {
  const t = useTranslations('TermsNotice')
  
  return (
    <p>
      {t.rich('signUpAgreement', {
        terms: (chunks) => <a href="/terms">{chunks}</a>
      })}
    </p>
  )
}
```

**Translation file:**
```json
{
  "TermsNotice": {
    "signUpAgreement": "By clicking Sign Up, you agree to our <terms>Terms of Service</terms>"
  }
}
```

#### Multiple Rich Elements

```tsx
<p>
  {t.rich('legalNotice', {
    privacy: (chunks) => <a href="/privacy">{chunks}</a>,
    terms: (chunks) => <a href="/terms">{chunks}</a>,
    bold: (chunks) => <strong>{chunks}</strong>
  })}
</p>
```

```json
{
  "legalNotice": "Read our <privacy>Privacy Policy</privacy> and <terms>Terms</terms>. <bold>Important:</bold> All rights reserved."
}
```

#### Rich Text with Dynamic Props

```tsx
<p>
  {t.rich('downloadLink', {
    link: (chunks) => (
      <a href={fileUrl} download className="text-primary-600">
        {chunks}
      </a>
    )
  })}
</p>
```

```json
{
  "downloadLink": "Click <link>here</link> to download the file"
}
```

### Rich Text with i18next (Trans Component)

#### Basic Trans Example

```tsx
import { Trans } from 'react-i18next'

function Welcome() {
  return (
    <p>
      <Trans i18nKey="welcomeMessage">
        Welcome to our app! Check out our <a href="/guide">guide</a> to get started.
      </Trans>
    </p>
  )
}
```

**Translation:**
```json
{
  "welcomeMessage": "Welcome to our app! Check out our <1>guide</1> to get started."
}
```

#### Trans with Named Components

```tsx
<Trans 
  i18nKey="legal"
  components={{
    termsLink: <a href="/terms" />,
    privacyLink: <a href="/privacy" />,
    bold: <strong />
  }}
/>
```

```json
{
  "legal": "By continuing, you agree to our <termsLink>Terms</termsLink> and <privacyLink>Privacy Policy</privacyLink>. <bold>Your data is protected.</bold>"
}
```

### Combining Rich Text with Pluralization

You can combine both features for powerful, flexible translations:

```tsx
<p>
  {t.rich('cartSummary', {
    count: items.length,
    price: totalPrice,
    link: (chunks) => <a href="/cart">{chunks}</a>,
    bold: (chunks) => <strong>{chunks}</strong>
  })}
</p>
```

```json
{
  "cartSummary": "{count, plural, =0 {Your cart is empty} one {You have # item in <link>your cart</link> • <bold>${price}</bold>} other {You have # items in <link>your cart</link> • <bold>${price}</bold>}}"
}
```

## Number and Currency Formatting

### Currency Example

```tsx
<p>{t('productPrice', { amount: 99.99, currency: 'USD' })}</p>
```

```json
{
  "productPrice": "Price: {amount, number, ::currency/{currency}}"
}
```

**Output by locale:**
- English (US): "Price: $99.99"
- German: "Price: 99,99 €"
- Japanese: "Price: ¥100"

### Number Formatting

```tsx
<p>{t('largeNumber', { num: 1234567.89 })}</p>
```

```json
{
  "largeNumber": "{num, number, ::compact-short}"
}
```

**Output by locale:**
- English: "1.2M"
- German: "1,2 Mio."

## Date and Time Formatting

### Date Example

```tsx
<p>{t('publishDate', { date: new Date('2024-01-15') })}</p>
```

```json
{
  "publishDate": "Published on {date, date, long}"
}
```

**Date format options:**
- `short`: 1/15/24
- `medium`: Jan 15, 2024
- `long`: January 15, 2024
- `full`: Monday, January 15, 2024

### Time Example

```tsx
<p>{t('eventTime', { time: new Date() })}</p>
```

```json
{
  "eventTime": "Event starts at {time, time, short}"
}
```

**Time format options:**
- `short`: 3:30 PM
- `medium`: 3:30:00 PM
- `long`: 3:30:00 PM UTC
- `full`: 3:30:00 PM Coordinated Universal Time

## Best Practices

### ✅ Do's

1. **Keep sentences together**: Use rich text instead of splitting sentences
   ```tsx
   // Good
   {t.rich('terms', { link: (c) => <a>{c}</a> })}
   
   // Bad
   {t('before')} <a>{t('link')}</a> {t('after')}
   ```

2. **Use descriptive placeholder names**:
   ```json
   {
     "message": "Contact <supportLink>our support team</supportLink>"
   }
   ```

3. **Provide context for translators**:
   ```json
   {
     "itemCount": "{count, plural, one {# item} other {# items}}",
     "_itemCount_context": "Shopping cart item counter"
   }
   ```

4. **Test all plural forms** in your target languages

5. **Use exact matches** for special cases:
   ```json
   {
     "days": "{count, plural, =0 {Today} =1 {Tomorrow} other {In # days}}"
   }
   ```

### ❌ Don'ts

1. **Don't hardcode counts in text**:
   ```tsx
   // Bad
   {count === 1 ? t('oneItem') : t('manyItems')}
   
   // Good
   {t('items', { count })}
   ```

2. **Don't split rich text unnecessarily**:
   ```tsx
   // Bad
   {t('text1')} <a>{t('link')}</a> {t('text2')}
   
   // Good
   {t.rich('message', { link: (c) => <a>{c}</a> })}
   ```

3. **Don't forget the `other` category** in plurals (it's required)

4. **Don't use `#` outside plural expressions** (it's specific to plurals)

## Workflow with i18nizer

1. **Run i18nizer** to extract base translations:
   ```bash
   i18nizer translate src/components/Cart.tsx --locales en,es,fr
   ```

2. **Identify patterns** that need pluralization or rich text

3. **Manually enhance** the extracted translations:
   - Replace ternary count logic with ICU plurals
   - Replace split sentences with rich text

4. **Update your component** to use the enhanced translation keys

5. **Test** with different locales and count values
